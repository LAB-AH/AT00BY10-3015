  1. Markdown HTML

    Create easy to read Markdown document of the topic of your own choice. Your document can be as simple or as complex as you decide.

  3. Integration testing

  4. UI testing
    1. Test any website with browser devtools.

  5. WEB GUI Testing

    Do a small "frontend" with your preferred framework.

  6. Logging
    1. Implement Logging library.
    2. Tally counter REST API.
  7. Versioning and Licensing
    1. Find the version number of any of your projects (on this course). This can be a commit version, or if you have not used versioning, you can retroactively describe a version number for any of your previous projects. Then read how semantic versioning works according to the semver.org.
    2. Write small license for your software.
  8. Deployment
    • Implement unit tests for the provided library.
    • You can use any tools you like, and implement the tests in any sane way you deem applicable.
    • Implement a working GitHub Actions workflow consisting your test repository, test runner and coverage reports (Coveralls).
    • Report any issues you found in your GitHub repository's issue tracker.
